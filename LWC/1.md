# Euler Problem 1 Solution in Lightning Web Component

Euler Problem 1 asks: "Find the sum of all the multiples of 3 or 5 below 1000."

Here's the solution implemented as a Lightning Web Component:

## JavaScript Controller (lwc/eulerProblem1/eulerProblem1.js)
```javascript
import { LightningElement } from 'lwc';

export default class EulerProblem1 extends LightningElement {
    result = 0;
    numbers = [];

    connectedCallback() {
        this.calculateSum();
    }

    calculateSum() {
        let sum = 0;
        const multiples = [];

        for (let i = 1; i < 1000; i++) {
            if (i % 3 === 0 || i % 5 === 0) {
                sum += i;
                multiples.push(i);
            }
        }

        this.result = sum;
        this.numbers = multiples;
    }

    get displayNumbers() {
        return this.numbers.slice(0, 20).join(', ') + 
               (this.numbers.length > 20 ? '...' : '');
    }
}
```

## Alternative Implementation (More Efficient)

For better performance, here's an optimized version:

```javascript
import { LightningElement } from 'lwc';

export default class EulerProblem1 extends LightningElement {
    result = 0;

    connectedCallback() {
        this.calculateSumOptimized();
    }

    calculateSumOptimized() {
        // Using arithmetic progression formula for better performance
        const sumOfMultiples = (divisor, limit) => {
            const n = Math.floor((limit - 1) / divisor);
            return divisor * n * (n + 1) / 2;
        };

        // Sum of multiples of 3 + Sum of multiples of 5 - Sum of multiples of 15
        // (to avoid double counting)
        this.result = sumOfMultiples(3, 1000) + 
                     sumOfMultiples(5, 1000) - 
                     sumOfMultiples(15, 1000);
    }
}
```

## Explanation

The solution works by:

1. **Brute Force Approach**: Check each number from 1 to 999 to see if it's divisible by 3 or 5
2. **Optimized Mathematical Approach**: Use the arithmetic progression formula to calculate sums directly
3. **Result**: The answer is **233168**

The optimized version is more efficient as it runs in O(1) time complexity instead of O(n), where n is 999.
