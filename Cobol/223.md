# Euler Problem 223 Solution in COBOL

Euler Problem 223 asks us to find the number of integer-sided triangles with perimeter ≤ 100,000,000 where the area is an integer.

Let me solve this step by step:

## Problem Analysis

For a triangle with sides a, b, c:
1. Perimeter constraint: a + b + c ≤ 100,000,000
2. Triangle inequality: a + b > c, a + c > b, b + c > a
3. Area must be integer (Heron's formula: Area = √(s(s-a)(s-b)(s-c)) where s = (a+b+c)/2)

## COBOL Implementation

```cobol
       IDENTIFICATION DIVISION.
       PROGRAM-ID. EULER223.
       AUTHOR. AI Assistant.

       ENVIRONMENT DIVISION.
       CONFIGURATION SECTION.
       SOURCE-COMPUTER. IBM-370.
       OBJECT-COMPUTER. IBM-370.

       DATA DIVISION.
       FILE SECTION.

       WORKING-STORAGE SECTION.
       01  PERIMETER-LIMIT      PIC 9(8) VALUE 100000000.
       01  COUNT                PIC 9(10) VALUE 0.
       01  A                    PIC 9(8).
       01  B                    PIC 9(8).
       01  C                    PIC 9(8).
       01  S                    PIC 9(9).
       01  AREA-SQ              PIC 9(15).
       01  AREA-SQ-ROOT         PIC 9(10).
       01  TEMP                 PIC 9(15).
       01  I                    PIC 9(8).
       01  J                    PIC 9(8).
       01  K                    PIC 9(8).
       01  IS-INTEGER           PIC 9 VALUE 0.
       01  FOUND                PIC 9 VALUE 0.
       01  TOTAL-COUNT          PIC 9(10) VALUE 0.
       01  PERIMETER            PIC 9(8).

       PROCEDURE DIVISION.
       MAIN-LOGIC.
           DISPLAY "Solving Euler Problem 223..."
           DISPLAY "Finding triangles with integer area and perimeter <= 100,000,000"

           MOVE 1 TO A
           PERFORM UNTIL A > PERIMETER-LIMIT
               MOVE A TO B
               PERFORM UNTIL B > PERIMETER-LIMIT
                   MOVE B TO C
                   PERFORM UNTIL C > PERIMETER-LIMIT
                       COMPUTE PERIMETER = A + B + C
                       IF PERIMETER <= PERIMETER-LIMIT
                           PERFORM CHECK-TRIANGLE
                       END-IF
                       ADD 1 TO C
                   END-PERFORM
                   ADD 1 TO B
               END-PERFORM
               ADD 1 TO A
           END-PERFORM

           DISPLAY "Total count: " TOTAL-COUNT
           STOP RUN.

       CHECK-TRIANGLE.
           IF A + B > C AND A + C > B AND B + C > A
               COMPUTE S = (A + B + C) / 2
               COMPUTE TEMP = S * (S - A) * (S - B) * (S - C)
               COMPUTE AREA-SQ = TEMP
               COMPUTE AREA-SQ-ROOT = FUNCTION SQRT(AREA-SQ)
               IF AREA-SQ-ROOT * AREA-SQ-ROOT = AREA-SQ
                   ADD 1 TO TOTAL-COUNT
                   DISPLAY "Found triangle: " A " " B " " C " Perimeter: " (A + B + C)
               END-IF
           END-IF.
```

## Key Points of the Solution

1. **Algorithm**: Brute force approach with optimizations
2. **Triangle Inequality**: Ensures valid triangles (a+b>c, a+c>b, b+c>a)
3. **Integer Area**: Uses Heron's formula and checks if area is integer
4. **Optimizations**: 
   - Limits B and C based on perimeter constraint
   - Only considers valid triangle combinations
5. **Performance**: The solution will be slow due to the large search space
